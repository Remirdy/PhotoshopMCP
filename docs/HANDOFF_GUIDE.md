# Developer Handoff & Specification Guide

This guide details the specifications and formats generated by the Photoshop MCP Handoff tools to facilitate developer transitions.

## 1. Geometric Specifications
Each layer's position is recorded as standard pixel offsets relative to the parent canvas origin (top-left, `0,0`):
- **X/Y coordinates**: Layer bounding box top-left offset.
- **Width/Height**: Actual rendering size.
- **Opacity**: Continuous decimal `0.0` to `1.0`.

## 2. Platform Conversion Principles

### CSS Stylesheet Output
Generated code maps layer styling directly to absolute CSS positioning layout classes:
```css
.layer-subject {
  position: absolute;
  left: 100px;
  top: 200px;
  width: 700px;
  height: 1000px;
  opacity: 1.0;
}
```

### SwiftUI Views Code
SwiftUI positioned image view blocks are generated with offset modifiers:
```swift
Image("Subject")
  .resizable()
  .frame(width: 700, height: 1000)
  .offset(x: 100, y: 200)
```

### Jetpack Compose Layouts
Compose positioned painter blocks are written with modifier sizes:
```kotlin
Image(
  painter = painterResource(id = R.drawable.subject),
  contentDescription = "Subject",
  modifier = Modifier
    .size(700.dp, 1000.dp)
    .offset(100.dp, 200.dp)
)
```

### Flutter Positioned Widgets
Positioned widgets are used in Stack view layers:
```dart
Positioned(
  left: 100.0,
  top: 200.0,
  child: Image.asset(
    'assets/subject.png',
    width: 700.0,
    height: 1000.0,
  ),
)
```

## 3. Design Token Synchronization
Design tokens are exported as JSON maps matching Style Dictionary configurations, allowing shared variables for colors, fonts, and borders.
